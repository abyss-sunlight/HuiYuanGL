# 设置密码页面弹窗化改造计划

## 📋 项目概述

将当前的 `set-password` 独立页面改造为在 `profile` 页面中弹窗显示的模式，保持原有的样式风格和用户体验。

## 🎯 改造目标

1. **用户体验优化**：用户无需跳转页面，在当前页面即可完成密码设置
2. **样式一致性**：保持与现有弹窗相同的设计风格
3. **功能完整性**：保留所有原有功能，无任何功能缺失
4. **代码复用**：最大化复用现有代码和样式

### 2. 功能迁移计划

#### 2.1 创建密码设置模块 (`profile.set-password.js`)

**迁移内容**：
- 手机号输入处理
- 验证码发送逻辑
- 倒计时功能
- 密码验证
- 表单提交

**新增功能**：
- 弹窗显示/隐藏控制
- 与父页面的数据通信

**模块结构**：
```javascript
// profile.set-password.js
const setPasswordModule = {
  // 弹窗数据
  setPasswordData: {
    showSetPasswordModal: false,
    phone: '',
    code: '',
    newPassword: '',
    codeSending: false,
    codeText: '获取验证码',
    submitting: false
  },

  // 弹窗控制方法
  showSetPassword() { /* 显示弹窗 */ },
  hideSetPassword() { /* 隐藏弹窗 */ },
  
  // 原有方法迁移
  onPhoneInput() { /* 手机号输入 */ },
  onCodeInput() { /* 验证码输入 */ },
  onNewPasswordInput() { /* 密码输入 */ },
  sendSmsCode() { /* 发送验证码 */ },
  startCountdown() { /* 倒计时 */ },
  submitPassword() { /* 提交密码 */ }
}
```

### 4. 交互流程设计

#### 4.1 弹窗触发流程

```
用户点击"设置密码" 
    ↓
显示弹窗 (showSetPasswordModal: true)
    ↓
自动填充用户手机号
    ↓
用户输入验证码和新密码
    ↓
点击确认设置
    ↓
提交成功后关闭弹窗
    ↓
显示成功提示
```

#### 4.2 错误处理流程

```
输入验证失败
    ↓
显示错误提示 (wx.showToast)
    ↓
保持弹窗开启状态
    ↓
用户重新输入
```

#### 4.3 网络请求处理

```
发送验证码请求
    ↓
显示加载状态 (codeSending: true)
    ↓
请求成功 → 开始倒计时
    ↓
请求失败 → 显示错误信息，重置状态
```
