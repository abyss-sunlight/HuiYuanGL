# 美睫美甲会员管理小程序项目概述

## 项目简介

本项目是一个专为美睫美甲门店设计的会员管理小程序系统，核心功能围绕会员充值、消费记录、美睫服务记录管理展开，**并非线上预约系统**。系统通过线下服务与线上管理相结合的方式，为门店提供完整的会员运营解决方案。

## 核心功能

### 1. 会员充值管理
- 支持多种充值金额和对应的折扣优惠
- 充值记录实时更新会员账户余额
- 灵活的折扣配置系统（充值折扣表）

### 2. 消费记录管理
- 详细记录会员的每一笔消费
- 区分充值和支出两种消费类型
- 实时计算和更新会员余额

### 3. 美睫服务记录
- 记录会员的美睫服务详情
- 包含款式、型号、长度、翘度等专业参数
- 服务历史追踪管理

### 4. 权限分级管理
- 基于角色的权限控制系统
- 不同角色拥有不同的操作权限
- 确保数据安全和业务流程规范

## 用户角色及权限

### 1. 游客（权限等级：4）
- **核心权限**：无数据查看权限
- **功能限制**：只能访问登录页面和美睫记录
- **业务定位**：潜在客户，需要联系店长进行线下充值升级

### 2. 会员（权限等级：3）
- **核心权限**：查看个人消费记录、美睫记录、账户余额
- **功能限制**：无法查看其他会员信息，无法修改折扣配置
- **业务定位**：系统的主要用户，享受门店服务

### 3. 员工（权限等级：2）
- **核心权限**：查看所有会员信息、消费记录、美睫记录，充值操作
- **功能限制**：无法修改折扣配置
- **业务定位**：门店服务人员，协助会员管理

### 4. 店长（权限等级：1）
- **核心权限**：拥有所有功能权限
- **特殊功能**：充值操作、折扣配置管理、会员状态管理
- **业务定位**：门店管理者，负责会员升级和系统配置

## 关键业务流程

### 核心闭环流程
```
游客联系店长线下充值 → 店长在后台操作为其升级会员 → 会员消费并记录
```

### 详细流程说明

1. **游客转化阶段**
   - 游客到店咨询或通过联系方式联系店长
   - 店长介绍会员制度和充值优惠
   - 游客选择充值金额并完成线下支付

2. **会员升级阶段**
   - 店长登录管理系统
   - 创建用户账号并设置会员权限
   - 录入充值金额并应用相应折扣
   - 生成会员号和初始账户信息

3. **服务消费阶段**
   - 会员到店享受美睫美甲服务
   - 员工记录服务详情（款式、参数等）
   - 系统自动扣除相应费用
   - 更新会员余额和消费记录

4. **数据管理阶段**
   - 店长可查看营业数据和会员统计
   - 员工可查看会员服务历史
   - 会员可查看个人消费记录和余额

## 核心数据库表结构

### 1. user（用户表）
- **用途**：存储所有用户基本信息和权限数据
- **关键字段**：
  - `phone`：手机号（唯一标识）
  - `permission_level`：权限等级（1-店长，2-员工，3-会员，4-游客）
  - `amount`：账户余额
  - `member_no`：会员号（唯一值）

### 2. consume_records（消费记录表）
- **用途**：记录所有充值和消费交易
- **关键字段**：
  - `phone`：会员手机号
  - `consume_amount`：消费金额
  - `consume_type`：消费类型（充值/支出）
  - `consume_item`：消费项目

### 3. eyelash_records（美睫记录表）
- **用途**：记录美睫服务的专业参数
- **关键字段**：
  - `phone`：会员手机号
  - `style`：款式（中文）
  - `model_number`：型号
  - `length`：睫毛长度
  - `curl`：翘度

### 4. recharge_discount（充值折扣表）
- **用途**：配置不同充值金额的折扣优惠
- **关键字段**：
  - `recharge_amount`：充值金额
  - `discount_rate`：折扣率
  - `discount_percentage`：折扣百分比
  - `is_active`：是否启用
