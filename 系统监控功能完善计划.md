# 系统监控功能完善计划

## 项目概述

基于现有的基础监控功能，完善系统监控模块，提供完整的系统状态监控和性能指标展示，集成到微信小程序的管理页面中。

## 现有功能分析

### 已有功能
1. **健康检查接口** (`/api/health`) - 基础服务状态检查
2. **业务统计** (`/api/stats`) - 会员总数、今日新增、活跃用户统计
3. **用户状态日志** - 用户账户变更审计记录

### 功能缺失
- 系统性能监控（CPU、内存、JVM指标）
- 接口性能监控
- 错误日志监控
- 实时监控数据展示

## 功能完善计划

### 后端功能扩展

#### 1. 系统性能监控服务
**文件路径**: `backend/src/main/java/com/example/membersystem/monitor/`

**新增文件**:
- `SystemMonitorService.java` - 系统性能数据收集
- `SystemMonitorController.java` - 监控数据API接口
- `dto/SystemMetrics.java` - 系统指标数据传输对象

**功能特性**:
```java
// 系统指标包含
- CPU使用率
- 内存使用情况（已用/总量）
- JVM堆内存信息
- 磁盘使用情况
- 系统运行时间
```

#### 2. 接口性能监控
**文件路径**: `backend/src/main/java/com/example/membersystem/monitor/`

**新增文件**:
- `ApiPerformanceInterceptor.java` - 接口性能拦截器
- `ApiPerformanceService.java` - 性能数据统计服务
- `entity/ApiPerformanceLog.java` - 性能日志实体

**监控指标**:
- 接口响应时间
- 请求频率统计
- 错误率统计
- 慢接口识别

#### 3. 监控数据缓存策略
**技术方案**: 使用 Redis 缓存监控数据

**缓存策略**:
- 系统性能数据：缓存 30 秒
- 业务统计数据：缓存 5 分钟  
- 接口性能数据：缓存 1 分钟

## 技术实现细节

### 后端技术栈
- **Spring Boot Actuator**: 基础监控端点
- **Micrometer**: 指标收集框架
- **Redis**: 监控数据缓存
- **JMX**: JVM指标获取

### 前端技术栈
- **微信小程序原生组件**: 页面开发
- **ECharts**: 图表展示（通过web-view集成）
- **定时器**: 数据自动刷新

## 权限设计

### 访问控制
- **店长权限（level=1）**: 可查看所有监控数据
- **员工权限（level=2）**: 可查看基础统计信息
- **会员权限（level=3）**: 无权限访问

### 数据脱敏
- 敏感系统信息隐藏
- 错误日志信息过滤
- 内部IP地址脱敏

## 监控指标阈值

### 系统性能阈值
```yaml
thresholds:
  cpu:
    warning: 70%    # 警告阈值
    critical: 90%   # 严重阈值
  memory:
    warning: 80%    # 警告阈值  
    critical: 95%   # 严重阈值
  disk:
    warning: 85%    # 警告阈值
    critical: 95%   # 严重阈值
```

### 接口性能阈值
```yaml
api_thresholds:
  response_time:
    normal: 500ms     # 正常响应时间
    slow: 2000ms      # 慢接口阈值
    timeout: 5000ms   # 超时阈值
  error_rate:
    warning: 5%       # 错误率警告
    critical: 15%     # 错误率严重
```

## 风险评估

### 性能风险
- **监控数据采集开销**: 通过异步处理和缓存降低影响
- **频繁刷新请求**: 通过前端缓存和请求合并优化

### 安全风险  
- **敏感信息泄露**: 实施权限控制和数据脱敏
- **监控接口滥用**: 添加访问频率限制
