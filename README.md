## 1. 用户认证系统

### 1.1 登录方式

#### 短信验证码登录
- 用户输入手机号
- 系统发送6位数字验证码（验证码发送逻辑在后端控制器实现）
  - **后端控制器**：`AuthController.java` 第 88-117 行（`sendSms` 方法）
  - **接口路径**：`POST /api/auth/send-sms`
  - **业务服务层**：`SmsCodeService.java`
    - 验证码发送逻辑：第 54-92 行（`sendCode` / `sendSmsCode` 方法）
    - 验证码生成器：第 190-194 行（`generateCode` 方法）
    - 测试验证方式：通过控制台日志输出验证码（第 62-64, 85-91 行）
- 验证码验证成功后生成JWT Token
- 支持新用户自动注册

#### 密码登录
- 支持手机号+密码登录
- 密码采用混合加密存储 (BCrypt + Argon2)
- 错误次数限制保护

#### 微信授权登录
- 微信小程序一键登录
- 获取用户微信头像和昵称
- 新用户需补充手机号信息

### 1.2 安全特性
- JWT Token 身份认证
- Token自动续期机制
- 登录状态持久化

### 1.3 密码管理
- 设置/重置密码
- 原手机号验证
- 新手机号绑定

## 2. 用户管理

### 2.1 权限分级体系

| 角色 | 权限等级 | 功能权限 |
|------|----------|----------|
| 店长 | 1 | 全部功能：用户管理、折扣设置、数据统计、AI分析 |
| 员工 | 2 | 查看成员列表、消费记录管理、查看统计 |
| 会员 | 3 | 查看个人记录、使用AI客服 |
| 游客 | 4 | 浏览首页、查看轮播内容 |

### 2.2 用户CRUD操作

#### 创建用户 (仅店长)
- 指定手机号、姓氏、性别
- 设置权限等级
- 自动生成会员编号

#### 查询用户
- 按手机号模糊查询
- 按姓氏查询
- 按权限等级筛选
- 组合条件搜索

#### 更新用户
- 修改个人资料 (头像、昵称、性别)
- 修改用户状态 (启用/禁用)
- 修改手机号 (需双重验证)

#### 删除用户 (仅店长)
- 软删除保护
- 关联数据清理

### 2.3 会员编号规则
- 自动生成唯一会员号
- 格式：系统自定义
- 支持通过会员号查询

## 9. 安全机制

### 9.1 认证安全

#### JWT Token
- 签名验证
- 过期时间控制
- 防篡改保护

#### 权限控制
- 注解式权限检查 `@RequirePermission`
- 接口级权限控制
- 数据级权限过滤

### 9.2 密码安全

#### 混合加密方案
- **第一层**: BCrypt 哈希
- **第二层**: Argon2 哈希
- **优势**: 兼顾安全性和性能

#### 加密流程
```
明文密码 → BCrypt哈希 → Argon2哈希 → 数据库存储
```

### 9.3 数据安全
- SQL注入防护 (JPA参数化查询)
- XSS攻击防护
- CSRF防护
- 敏感数据脱敏

### 9.4 传输安全
- HTTPS通信 (生产环境)
- Token头部传输
- 敏感信息加密
