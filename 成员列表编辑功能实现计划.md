# 成员列表编辑功能实现计划

## 功能概述
为成员列表添加编辑功能，通过弹窗形式展示用户信息，并提供禁用/启用滑动块控制。

## 功能需求

### 1. 触发方式
- 点击成员列表中的编辑按钮触发弹窗
- 按钮已添加完成，无需额外开发

### 2. 弹窗界面设计
- **用户信息展示**：
  - 用户头像
  - 用户昵称
  - 用户角色
  - 联系方式（如有）
  - 注册时间
  - 最后活跃时间
- **展示要求**：
  - 信息紧凑显示，无输入框
  - 采用纯文本展示方式
  - 布局简洁明了

### 3. 编辑功能
- **核心功能**：禁用/启用滑动块
- **滑动块状态**：
  - 开启状态：用户正常使用
  - 关闭状态：用户被禁用
- **状态切换**：
  - 实时响应用户操作
  - 提供状态变更确认提示

## 技术实现方案

### 1. 前端实现
#### 1.1 弹窗组件
- 创建 `memberEditModal` 组件
- 支持显示/隐藏动画
- 响应式设计适配不同屏幕

#### 1.2 用户信息展示
```javascript
// 数据结构示例
const userInfo = {
  avatar: "头像URL",
  nickname: "用户昵称",
  role: "管理员/普通用户",
  phone: "联系电话",
  registerTime: "2024-01-01",
  lastActiveTime: "2024-03-26",
  status: "active/inactive"
}
```

#### 1.3 状态滑动块
- 使用微信小程序原生 `switch` 组件
- 绑定状态变更事件
- 实时同步到后端

### 2. 后端实现
#### 2.1 API接口设计
```
PUT /api/members/{userId}/status
参数：
- status: boolean (true=启用, false=禁用)
返回：
- success: boolean
- message: string
- data: 更新后的用户信息
```

#### 2.2 数据库更新
- 更新用户表中的状态字段
- 记录状态变更日志
- 处理禁用用户的相关业务逻辑

### 3. 状态管理
#### 3.1 前端状态
- 本地状态实时更新
- 列表页面状态同步
- 错误处理和重试机制

#### 3.2 后端状态
- 数据库事务处理
- 状态变更记录
- 权限验证

## UI设计规范

### 1. 弹窗布局
- 宽度：屏幕宽度的90%，最大宽度400px
- 高度：自适应内容，最大高度80%屏幕
- 圆角：8px
- 阴影：轻微阴影效果

### 2. 信息展示区域
- 头像：60x60px圆形
- 昵称：16px，加粗
- 其他信息：14px，常规
- 行间距：8px

### 3. 滑动块样式
- 颜色：启用-绿色，禁用-灰色
- 大小：标准尺寸
- 位置：信息区域下方

## 注意事项

### 1. 用户体验
- 弹窗显示时背景遮罩

- 防抖处理状态变更请求
