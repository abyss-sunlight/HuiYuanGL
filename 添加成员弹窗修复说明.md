# 添加成员弹窗修复说明

## 问题描述
成员列表里的添加成员弹窗中，已经输入了手机号但仍然提示"请输入手机号"。

## 问题原因
在 `members.wxml` 文件中，添加成员表单的手机号和姓氏输入框绑定了错误的事件处理方法：
- 手机号输入框绑定：`bindinput="onPhoneInput"`
- 姓氏输入框绑定：`bindinput="onLastNameInput"`

但这两个方法实际上是用来处理页面顶部筛选条件的输入，而不是添加成员表单的输入。因此，用户在添加成员弹窗中输入内容时，数据并没有正确更新到 `addMemberForm` 对象中。

## 修复内容

### 1. 添加专门的输入处理方法
在 `members.js` 文件中添加了两个新的方法：

```javascript
/**
 * 添加成员表单手机号输入处理
 */
onAddMemberPhoneInput(e) {
  this.setData({
    'addMemberForm.phone': e.detail.value
  })
},

/**
 * 添加成员表单姓氏输入处理
 */
onAddMemberLastNameInput(e) {
  this.setData({
    'addMemberForm.lastName': e.detail.value
  })
},
```

### 2. 更新WXML绑定事件
在 `members.wxml` 文件中更新了输入框的绑定事件：

```xml
<!-- 手机号 -->
<input class="form-input" type="number" placeholder="请输入手机号" 
       value="{{addMemberForm.phone}}" bindinput="onAddMemberPhoneInput" maxlength="11" />

<!-- 姓氏 -->
<input class="form-input" type="text" placeholder="请输入姓氏" 
       value="{{addMemberForm.lastName}}" bindinput="onAddMemberLastNameInput" />
```

## 修复后的数据流
1. 用户在添加成员弹窗中输入手机号
2. 触发 `onAddMemberPhoneInput` 事件
3. 更新 `data.addMemberForm.phone` 的值
4. 表单验证时能正确读取到输入的手机号
5. 验证通过，成功提交表单
