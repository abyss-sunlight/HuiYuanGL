# 微信注册响应格式问题修复

## 🎯 **问题描述**

用户反馈：**微信注册响应包含完整的用户数据，但页面提示注册失败**

### **问题现象**
```
微信注册响应: {token: "eyJhbGciOiJIUzI1NiJ9...", userId: 17, username: "微信用户", lastName: "高", gender: 1, …}
响应状态码: undefined
响应数据: undefined
响应token: undefined
注册失败，响应码: undefined 消息: undefined
```

### **根本原因**
后端直接返回了`LoginResponse`对象，而不是被`ApiResponse`包装的格式，但前端代码期望的是`{code: 0, data: LoginResponse}`格式。

## 🔍 **问题分析**

### **后端返回格式**
```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "userId": 17,
  "username": "微信用户",
  "lastName": "高",
  "gender": 1,
  "permissionLevel": 3,
  "amount": 0,
  "discount": 100,
  "createdAt": "2026-03-22T10:19:00"
}
```

### **前端期望格式**
```json
{
  "code": 0,
  "message": "OK",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "userId": 17,
    "username": "微信用户",
    "lastName": "高",
    "gender": 1,
    "permissionLevel": 3,
    "amount": 0,
    "discount": 100,
    "createdAt": "2026-03-22T10:19:00"
  }
}
```

### **数据流问题**
```
后端: LoginResponse对象 → Spring序列化 → 直接JSON对象
     ↓
前端request.js: 期望ApiResponse格式 → 解析失败 → undefined
     ↓
前端profile.js: 期望response.code和response.data → 解析失败
```

## 🛠️ **修复方案**

### **1. 修改request.js处理逻辑**

#### **添加直接业务数据检测**
```javascript
// 检查是否是直接返回的业务数据（如LoginResponse）
if (body && typeof body === 'object' && body.token) {
  // 直接返回业务数据（这种情况发生在后端直接返回LoginResponse等对象）
  resolve(body)
  return
}
```

#### **完整处理逻辑**
```javascript
success(res) {
  // 后端约定返回：{ code, message, data } 或直接返回业务数据
  const body = res.data
  
  // 检查是否是直接返回的业务数据（如LoginResponse）
  if (body && typeof body === 'object' && body.token) {
    // 直接返回业务数据
    resolve(body)
    return
  }
  
  // 检查是否是ApiResponse包装的格式
  if (body && body.code === 0) {
    resolve(body.data)
    return
  }
  
  // 处理微信登录新用户需要补充信息的情况
  if (body && body.code === 201) {
    resolve(body)
    return
  }
  
  // 其他处理...
}
```

### **2. 修改profile.js响应处理**

#### **简化响应处理**
```javascript
}).then(response => {
  wx.hideLoading()
  console.log('微信注册响应:', response)
  console.log('响应类型:', typeof response)
  
  // 检查响应是否包含token（直接返回LoginResponse的情况）
  if (response && response.token) {
    // 注册成功，直接使用响应作为用户信息
    const userInfo = response
    wx.setStorageSync('userInfo', userInfo)
    wx.setStorageSync('token', response.token || '')
    
    // 关闭弹窗
    this.hideWxRegisterModal()
    
    wx.showToast({
      title: '注册成功',
      icon: 'success'
    })
    
    // 刷新页面状态
    this.checkLoginStatus()
  } else if (response && response.code === 201) {
    // 新用户需要补充信息的情况
    console.log('新用户需要补充信息')
    wx.showToast({
      title: '需要补充信息',
      icon: 'none'
    })
  } else {
    console.log('注册失败，响应:', response)
    wx.showToast({
      title: '注册失败',
      icon: 'none'
    })
  }
})
```
