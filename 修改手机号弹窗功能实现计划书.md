# 修改手机号弹窗功能实现计划书

## 📋 项目概述

参考现有的设置密码弹窗功能，在 `profile` 页面中添加一个修改手机号的弹窗功能，包括前端弹窗交互、后端API接口和数据库更新操作。

## 🎯 功能目标

1. **用户体验优化**：用户无需跳转页面，在当前页面即可完成手机号修改
2. **安全性保障**：通过原手机号验证码和新手机号验证码双重验证
3. **样式一致性**：保持与现有密码弹窗相同的设计风格
4. **功能完整性**：包括验证码发送、手机号验证、数据更新等完整流程

#### 2.3 扩展认证服务

**修改 AuthService.java**：
```java
/**
 * 发送原手机号验证码
 * 验证手机号是否属于当前用户
 */
public void sendOriginalSms(String phone) {
    // 验证手机号是否属于当前登录用户
    User currentUser = getCurrentUser();
    if (!phone.equals(currentUser.getPhone())) {
        throw new BusinessException(400, "原手机号不匹配");
    }
    
    // 发送验证码
    smsCodeService.sendSmsCode(phone, SmsCodeService.Type.CHANGE_PHONE_ORIGINAL);
}

/**
 * 发送新手机号验证码
 * 验证新手机号是否已被使用
 */
public void sendNewSms(String phone) {
    // 检查新手机号是否已被使用
    if (userRepository.existsByPhone(phone)) {
        throw new BusinessException(400, "该手机号已被使用");
    }
    
    // 发送验证码
    smsCodeService.sendSmsCode(phone, SmsCodeService.Type.CHANGE_PHONE_NEW);
}

/**
 * 修改手机号
 * 通过双重验证码验证后更新手机号
 */
@Transactional
public void changePhone(ChangePhoneRequest request) {
    // 验证原手机号验证码
    smsCodeService.verifyCode(
        request.getOriginalPhone(), 
        request.getOriginalCode(), 
        SmsCodeService.Type.CHANGE_PHONE_ORIGINAL
    );
    
    // 验证新手机号验证码
    smsCodeService.verifyCode(
        request.getNewPhone(), 
        request.getNewCode(), 
        SmsCodeService.Type.CHANGE_PHONE_NEW
    );
    
    // 获取当前用户
    User currentUser = getCurrentUser();
    
    // 验证原手机号匹配
    if (!request.getOriginalPhone().equals(currentUser.getPhone())) {
        throw new BusinessException(400, "原手机号不匹配");
    }
    
    // 更新手机号
    currentUser.setPhone(request.getNewPhone());
    userRepository.save(currentUser);
    
    // 清除相关验证码
    smsCodeService.clearCode(request.getOriginalPhone());
    smsCodeService.clearCode(request.getNewPhone());
}
```

### 3. 数据库设计

#### 3.1 现有结构分析
当前 `user` 表的 `phone` 字段已经支持修改：
- `phone VARCHAR(32) NOT NULL UNIQUE` - 唯一约束，支持更新
- `INDEX idx_phone (phone)` - 索引，自动维护

#### 3.2 数据一致性保障
```sql
-- 手机号修改时的数据一致性检查
DELIMITER //
CREATE PROCEDURE check_phone_change(
    IN user_id BIGINT,
    IN old_phone VARCHAR(32),
    IN new_phone VARCHAR(32)
)
BEGIN
    DECLARE current_phone VARCHAR(32);
    DECLARE phone_exists INT;
    
    -- 获取当前手机号
    SELECT phone INTO current_phone FROM user WHERE id = user_id;
    
    -- 验证原手机号匹配
    IF current_phone != old_phone THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '原手机号不匹配';
    END IF;
    
    -- 检查新手机号是否已存在
    SELECT COUNT(*) INTO phone_exists FROM user 
    WHERE phone = new_phone AND id != user_id;
    
    IF phone_exists > 0 THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = '新手机号已被使用';
    END IF;
    
    -- 更新手机号
    UPDATE user SET phone = new_phone WHERE id = user_id;
END //
DELIMITER ;
```

## ⚠️ 注意事项

### 安全注意事项
1. **双重验证**：必须验证原手机号和新手机号
2. **验证码有效期**：建议5-10分钟有效期
3. **频率限制**：同一手机号1分钟内只能发送1次
4. **并发控制**：防止同一用户重复修改

### 用户体验注意事项
1. **自动填充**：原手机号自动从用户信息获取
2. **实时验证**：手机号格式实时检查
3. **状态反馈**：每个操作都有明确的加载状态
4. **错误提示**：友好的中文错误信息

### 技术注意事项
1. **事务处理**：手机号修改需要数据库事务
2. **数据一致性**：确保手机号唯一性约束
3. **缓存更新**：修改后更新用户信息缓存
4. **日志记录**：记录手机号修改操作日志
