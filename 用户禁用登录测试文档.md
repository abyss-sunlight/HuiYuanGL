# 用户禁用登录测试文档

## 问题描述
用户被禁用后，短信登录会反馈服务器错误而不是明确的"账号被禁用"提示。

## 解决方案
修改了所有登录服务的异常处理，确保被禁用用户登录时返回明确的错误信息。

## 修改内容

### 1. SmsLoginService 修改
- **文件**: `backend/src/main/java/com/example/membersystem/auth/service/SmsLoginService.java`
- **修改**: 将 `validateUserStatus` 方法中的 `RuntimeException` 改为 `BusinessException(40101, "账户已被禁用，请联系管理员")`
- **效果**: 被禁用用户短信登录时返回明确的错误信息

### 2. PasswordLoginService 修改
- **文件**: `backend/src/main/java/com/example/membersystem/auth/service/PasswordLoginService.java`
- **状态**: 已经正确使用 `BusinessException(40003, "账户已被禁用")`
- **效果**: 被禁用用户密码登录时返回明确的错误信息

### 3. WxLoginService 修改
- **文件**: `backend/src/main/java/com/example/membersystem/auth/service/WxLoginService.java`
- **修改**: 
  - 添加了 `validateUserStatus` 方法
  - 在用户登录流程中调用状态验证
  - 使用 `BusinessException(40101, "账户已被禁用，请联系管理员")`
- **效果**: 被禁用用户微信登录时返回明确的错误信息

### 4. AuthController 修改
- **文件**: `backend/src/main/java/com/example/membersystem/auth/controller/AuthController.java`
- **修改**: 移除了 `smsLogin` 和 `passwordLogin` 方法中的 try-catch 块，让 `BusinessException` 被 `GlobalExceptionHandler` 正确处理
- **效果**: 统一的错误响应格式

## 错误码说明

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| 40101 | "账户已被禁用，请联系管理员" | 用户状态为禁用时返回 |
| 40003 | "账户已被禁用" | 密码登录时用户状态为禁用时返回 |

## 全局异常处理

`GlobalExceptionHandler` 会根据 `BusinessException` 的错误码返回相应的HTTP状态码：

- **40101**: 返回 `401 Unauthorized` 状态码
- **40003**: 返回 `400 Bad Request` 状态码

## 前端处理建议

前端应该根据错误码和错误信息进行相应的处理：

```javascript
// 示例前端处理逻辑
if (response.code === 40101 || response.code === 40003) {
  // 显示账户被禁用的提示
  wx.showToast({
    title: '账户已被禁用，请联系管理员',
    icon: 'none',
    duration: 3000
  });
  // 跳转到客服页面或帮助页面
  wx.navigateTo({
    url: '/pages/customer-service/index'
  });
}
```
