# 微信登录新用户弹窗修复测试说明

## 问题描述
微信登录为新用户时，没有弹窗补充性别姓氏弹窗。

## 问题原因
后端 `WxLoginService.java` 中，当新用户缺少必要信息时，抛出的是 `RuntimeException` 而不是 `BusinessException(201)`，导致前端无法识别这是"需要补充信息"的情况。

## 修复内容
1. **修复异常类型**：将 `WxLoginService.java` 第83行的 `RuntimeException` 改为 `BusinessException(201)`
2. **添加导入**：在文件顶部添加 `import com.example.membersystem.common.BusinessException;`

## 修复后的代码逻辑
```java
if (!StringUtils.hasText(request.getUserInfo().getLastName())) {
    // 新用户但缺少必要信息，返回特殊状态码
    System.out.println("新用户缺少必要信息，需要补充");
    throw new BusinessException(201, "新用户需要补充姓氏和性别信息");
}
```

### 4. 验证前端响应
- 检查后端日志，应该看到 "新用户缺少必要信息，需要补充" 的输出
- 检查前端网络请求，应该收到状态码为201的响应
- 确认弹窗 `showWxRegisterModal` 被设置为 `true`

## 预期效果
修复后，新用户通过微信登录时：
1. 后端检测到缺少姓氏信息
2. 抛出 `BusinessException(201)` 
3. `AuthController` 捕获异常并返回状态码201的响应
4. 前端 `profile.wechat.js` 接收到201响应
5. 显示微信新用户注册弹窗
